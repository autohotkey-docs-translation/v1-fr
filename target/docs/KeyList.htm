Souris

Boutons Générales

Name Description
LButton Le bouton gauche de la souris lorsqu'il est utilisé avec Send, mais le bouton principal de la souris lorsqu'il est utilisé avec des hotkeys. Autrement dit, si l'utilisateur a échangé les boutons via les paramètres système, LButton :: est activé physiquement en cliquant sur le bouton droit de la souris, mais Send {LButton} effectue la même chose que de cliquer physiquement sur le bouton gauche. Pour toujours effectuer un clic gauche logiquement, utilisez Click Left ou Send {Click Left}.
RButton Le bouton droit de la souris lorsqu'il est utilisé avec Send, mais le bouton secondaire de la souris lorsqu'il est utilisé avec des hotkeys. Autrement dit, si l'utilisateur a échangé les boutons via les paramètres système, RButton :: est activé physiquement en cliquant sur le bouton gauche de la souris, mais Send {RButton} effectue la même chose que de cliquer physiquement sur le bouton droit. Pour toujours effectuer un clic droit logiquement, utilisez Click Right ou Send {Click Right}.
MButton Bouton central ou roulette de la souris

[v1.0.48+]: Faites défiler vers la gauche ou la droite.

Nécessite Windows Vista ou une version ultérieure. Ceux-ci peuvent être utilisés comme hotkey avec certaines souris (mais pas toutes) qui ont une deuxième roulette ou qui supportent l'inclinaison de la roulette de chaque côté. Clavier

Remarque: Les noms des touches de saisie de texte (alphabétiques et numériques) sont les mêmes que la lettre ou le chiffre comme tel. Par exemple : b is B and 5 is 5.

Bien que n'importe quel caractère unique puisse être utilisé comme nom de touche, sa signification (code de balayage ou code de touche virtuel) dépend de la disposition actuelle du clavier. En outre, certains caractères spéciaux doivent être échappés ou placés entre accolades, selon le contexte. [v1.1.27+]: Les lettres a-z ou A-Z peuvent être utilisées pour désigner les codes de touches virtuels correspondants (généralement vk41-vk5A) même s'ils ne sont pas inclus dans la disposition actuelle du clavier.

Spécifiez pour nn le code hexadécimal de la touche virtuelle d'une touche. Cette méthode rarement utilisée empêche également certains types de hotkeys d'exiger le hook clavier. Par exemple, le hotkey suivant n'utilise pas le hook clavier, mais comme effet secondaire, il est déclenché en appuyant sur soit Début ou PavnumDébut:

^VK24::MsgBox Vous avez appuyé sur Début ou PavnumDébut tout en maintenant la touche Ctrl enfoncée.

Limitation connue: Les Hotkeys VK doivent utiliser le hook clavier, comme * VK24 ou ~ VK24, va donc déclancher pour seulement une des touches, pas les deux (ex. PavnumDébut mais pas Début). Pour plus d'informations sur la méthode VKnn, veulliez voir Touches spéciales.

[v1.1.27+]: La présence d'un suffixe invalide empêche VKnn d'être reconnu. Par exemple, vk1Bsc001:: génère une erreur dans v1.1.27+, mais sc001 a été ignoré (sans effet) dans les versions précédentes.

Joystick

Joy1 à Joy32: Les boutons du joystick. Pour vous aider à déterminer les numéros de boutons de votre joystick, utilisez ce script de test. Note that hotkey prefix symbols such as ^ (control) and + (shift) are not supported (though GetKeyState() can be used as a substitute). Notez également que la détection des boutons du joystick "passe à travers" à la fenêtre active si cette fenêtre est conçue pour détecter la pression des boutons du joystick.

Although the following Joystick control names cannot be used as hotkeys, they can be used with GetKeyState():

  • JoyX, JoyY et JoyZ: ​​Les axes X (horizontal), Y (vertical) et Z (altitude / profondeur) du joystick.
  • JoyR: Le gouvernail ou le 4ème axe du joystick.
  • JoyU et JoyV: Les 5ème et 6ème axes du joystick.
  • JoyPOV: Le contrôle du point-de-vue (chapeau).
  • JoyName: Le nom du joystick ou de son pilote.
  • JoyButtons: Le nombre de boutons pris en charge par le joystick (pas toujours précis).
  • JoyAxes: Le nombre d'axes supportés par le joystick.
  • JoyInfo: Fournit une chaîne de texte composée de zéro ou plus des lettres suivantes pour indiquer les capacités du joystick: Z (a l'axe Z), R (a l'axe R), U (a l'axe U), V (a l'axe V), P (a la contrôle point-de-vue / POV), D (La contrôle point-de-vue / POV a un nombre limité de paramètres discrètes/distincts), C (la contrôle point-de-vue / POV est continu/fin). Exemple de chaîne de texte: ZRUVPD

Joysticks multiples: Si l'ordinateur possède plus d'un joystick et que vous voulez en utiliser un au-delà du premier, incluez le numéro du joystick (max 16) devant le nom du contrôle. Par exemple, 2joy1 est le premier bouton du deuxième joystick.

Remarque: Si vous rencontrez des difficultés pour qu'un script reconnaisse votre joystick, une personne a signalé le besoin de spécifier un numéro de joystick autre que 1 même si un seul joystick était présent. On ne sait pas comment cette situation se produit ou si elle est normale, mais vous pouvez expérimenter le numéro du joystick dans le script de test du joystick pour aider à déterminer si cela s'applique à votre système.

Voir aussi:

  • Joystick remapping: Methods of sending keystrokes and mouse clicks with a joystick.
  • Script Joystick-To-Mouse: Using a joystick as a mouse.

Touches Spéciales

Si votre clavier ou souris possède une touche qui ne se trouve pas dans la liste ci-dessus, vous pourrez malgré peut-être en faire un hotkey en procédant comme suit:

  1. Assurez-vous qu'au moins un script en cours d'exécution utilise le hook clavier. Vous pouvez déterminer si un script utilise le hook clavier en ouvrant sa fenêtre principale et en sélectionnant "View->Key History" dans la barre de menu.
  2. Double-cliquez sur l'icône de la barre d'état de ce script pour ouvrir sa fenêtre principale.
  3. Appuyez sur l'une des «touches mystère» de votre clavier.
  4. Sélectionnez l'élément de menu "View->Key History"
  5. Faites défiler vers le bas de la page. Quelque part vers le bas se trouvent les événements touche enfoncée (key-down) et touche relâchée (key-up) pour votre clé / touche. REMARQUE: Certaines touches ne génèrent pas d'événements et ne seront donc pas visibles ici. Si tel est le cas, vous ne pouvez pas directement faire de cette touche particulière un hotkey car votre pilote de clavier ou votre matériel le gère à un niveau trop bas pour que AutoHotkey aye accès. Pour des solutions possibles, veulliez voir ci-dessous.
  6. Si votre touche est détectable, notez la valeur hexadécimale à 3 chiffres dans la deuxième colonne de la liste (par exemple 159).
  7. Pour définir cette touche en tant que raccourci clavier, suivez cet exemple: 
    SC159:: ; Remplace 159 avec la valeur de votre touche.
MsgBox, %A_ThisHotkey% a été enfoncé.
return

Inverser la direction: Pour reconfigurer une autre touche afin de devenir une "touche mystère", suivez cet exemple:

; Remplace 159 avec la valeur découverte auparavant. Remplacez FF (si nécessaire) par le
; la touche virtuel de la touche, ce qui peut être découverte la première colonne dans l'écran Key History (l'Historique des touches).
#c::Send {vkFFsc159} ; Veulliez voir Send {vkXXscYYY} pour plus de détailes.

Solutions alternatives: si votre touche ou le bouton de la souris n'est pas détectable par l'écran Historique des touches, l'une des solutions suivantes peut vous aider:

  1. Reconfigurez le logiciel fourni avec votre souris ou votre clavier (parfois accessible dans le Panneau de configuration ou le menu Démarrer) pour que la "touche mystère" envoie une autre frappe. Ces frappes peuvent ensuite être définies comme hotkey dans un script. For example, if you configure a mystery key to send Ctrl+F1, you can then indirectly make that key as a hotkey by using ^F1:: in a script.

  2. Essayez AHKHID. Vous pouvez également essayer de rechercher sur le forum des mots clés tels que RawInput *, USB HID ou AHKHID.

  3. Ce qui suit est un dernier recours et ne doit généralement être tenté qu'en cas de désespoir. En effet, les chances de succès sont faibles et peuvent entraîner des effets secondaires indésirables difficiles à annuler:
    Désactivez ou supprimez tout logiciel supplémentaire fourni avec votre clavier ou votre souris ou remplacez son pilote par un pilote plus standard, tel que celui intégré au système d'exploitation. Cela suppose qu'il existe un tel pilote pour votre clavier ou souris et que vous pouvez vivre sans les fonctionnalités fournies par son pilote et son logiciel personnalisés.

VerrMaj et IME

Certaines configurations de Windows IME (telles que l'entrée japonaise avec un clavier anglais) utilisent VerrMaj pour basculer entre les modes. Dans de tels cas, VerrMaj est supprimé par l'IME et ne peut pas être détecté par AutoHotkey. Cependant, les touches Alt+Verr Maj, Ctrl+Verr Maj et Maj+Verr Maj raccourcis peuvent être désactiver avec une solution de contournement. Plus précisément, envoyez une touche (key-up) pour modifier l'état de l'IME, mais évitez tout autre effet en signalant au hook clavier de supprimer l'événement. La fonction suivante peut être utilisée à cet effet:

; Requère AutoHotkey v1.1.26+, et le hook clavier doit être installé.
#InstallKeybdHook
SendSuppressedKeyUp(key) {
    DllCall("keybd_event"
        , "char", GetKeyVK(key)
        , "char", GetKeySC(key)
        , "uint", KEYEVENTF_KEYUP := 0x2
        , "uptr", KEY_BLOCK_THIS := 0xFFC3D450)
}

Après avoir copié la fonction dans un script ou l'avoir enregistrée sous SendSuppressedKeyUp.ahk dans une bibliothèque de fonctions, ceci peut être utilisée comme suit:

; Désactiver les raccourcis Alt+key pour le IME.
~LAlt::SendSuppressedKeyUp("LAlt")

; Teste hotkey:
!CapsLock::MsgBox % A_ThisHotkey

; Remappez VerrMaj à Ctrl (gauche) d'une façon compatible avec IME.
*CapsLock::
    Send {Blind}{LCtrl DownR}
    SendSuppressedKeyUp("LCtrl")
    return
*CapsLock up::
    Send {Blind}{LCtrl Up}
    return
